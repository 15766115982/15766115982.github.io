2.C# PageFactory

  1. C#类中的PageFactory是页面对象设计模式的扩展。它是 Selenium WebDriver 的内置 POM 概念，但经过了非常优化。它用于初始化页面对象的元素或实例化页面对象本身。也可以创建（并推荐）元素的注释，因为描述属性可能并不总是足以描述一个对象与另一个对象。
    它用于初始化Page 类的元素，而不必使用“ FindElement ”或“ FindElements ”。注释可用于提供目标对象的描述性名称以提高代码可读性。
    PageObjects是Selenium 中的一种对象设计模式，其中网页被表示为类。页面上的各种元素被定义为类上的变量。然后，所有可能的用户交互都可以作为类上的方法来实现。由于类中命名良好的方法易于阅读，因此这是一种实现测试例程的优雅方式，这些例程既可读又易于维护或在未来更新。

  2. @FindBy 注解
    顾名思义，它有助于使用By策略 在页面中查找元素。@FindBy 可以接受TagName、PartialLinkText、Name、LinkText、Id、Css、ClassName、XPath 作为属性。用于定位元素或元素列表的替代机制。这允许用户快速轻松地创建 PageObjects。
    下面面的代码将创建一个PageObject并将其命名为UserName，方法是使用其ID定位器找到它。 [FindsBy(How = How.Id, Using = "username")]
    private IWebElement UserName { get; set; }

  3. 初始化元素
    实例化给定类的实例。此方法将尝试实例化给它的类，最好使用将 WebDriver 实例作为其唯一参数的构造函数，或者使用无参数构造函数。如果无法实例化该类，则会引发异常。
    WebDriver - 用于查找元素的驱动程序
    PageObjects - 将被初始化的类
    PageFactory.InitElements(WebDriver, PageObject);

  4. PageFactory 命名空间
    使用PageFactory需要引入DoNetSeleniumExtras.PageObjects库，并引用以下命名空间：
    using SeleniumExtras.PageObjects;;

  5. 实战

实战遇到的问题

1.PageFactory.InitElements()初始化报错

报错结果如下：

root casue:
Selenium.WebDriver、Selenium.Support是4.1.1版本的，但是DotNetSeleniumExtras.PageObjects是3.11.0版本的。
谷歌看了很多大佬的介绍，试了很多方法，发现降低Selenium.WebDriver、Selenium.Support、DotNetSeleniumExtras.PageObjects为同一版本可解决此问题。

2.Xpath定位元素出错

报错结果如下：

root cause：
是因为我在Xpath定位中用了”//input[contains(.,’pass’)]”，因为 . 是匹配当前节点下的所有属性，而页面中这个节点具有很多同名的，所以出错了，要匹配具体属性才行，比如”//input[contains(@id,’pass’)]”.
