2.C# PageFactory

  1. C#类中的PageFactory是页面对象设计模式的扩展。它是 Selenium WebDriver 的内置 POM 概念，但经过了非常优化。它用于初始化页面对象的元素或实例化页面对象本身。也可以创建（并推荐）元素的注释，因为描述属性可能并不总是足以描述一个对象与另一个对象。
    它用于初始化Page 类的元素，而不必使用“ FindElement ”或“ FindElements ”。注释可用于提供目标对象的描述性名称以提高代码可读性。
    PageObjects是Selenium 中的一种对象设计模式，其中网页被表示为类。页面上的各种元素被定义为类上的变量。然后，所有可能的用户交互都可以作为类上的方法来实现。由于类中命名良好的方法易于阅读，因此这是一种实现测试例程的优雅方式，这些例程既可读又易于维护或在未来更新。

  2. @FindBy 注解
    顾名思义，它有助于使用By策略 在页面中查找元素。@FindBy 可以接受TagName、PartialLinkText、Name、LinkText、Id、Css、ClassName、XPath 作为属性。用于定位元素或元素列表的替代机制。这允许用户快速轻松地创建 PageObjects。
    下面面的代码将创建一个PageObject并将其命名为UserName，方法是使用其ID定位器找到它。 [FindsBy(How = How.Id, Using = "username")]
    private IWebElement UserName { get; set; }

  3. 初始化元素
    实例化给定类的实例。此方法将尝试实例化给它的类，最好使用将 WebDriver 实例作为其唯一参数的构造函数，或者使用无参数构造函数。如果无法实例化该类，则会引发异常。
    WebDriver - 用于查找元素的驱动程序
    PageObjects - 将被初始化的类
    PageFactory.InitElements(WebDriver, PageObject);

  4. PageFactory 命名空间
    使用PageFactory需要引入DoNetSeleniumExtras.PageObjects库，并引用以下命名空间：
    using SeleniumExtras.PageObjects;;

  5. 实战

实战遇到的问题

1.PageFactory.InitElements()初始化报错

报错结果如下：

root casue:
Selenium.WebDriver、Selenium.Support是4.1.1版本的，但是DotNetSeleniumExtras.PageObjects是3.11.0版本的。
谷歌看了很多大佬的介绍，试了很多方法，发现降低Selenium.WebDriver、Selenium.Support、DotNetSeleniumExtras.PageObjects为同一版本可解决此问题。

2.Xpath定位元素出错

报错结果如下：

root cause：
是因为我在Xpath定位中用了”//input[contains(.,’pass’)]”，因为 . 是匹配当前节点下的所有属性，而页面中这个节点具有很多同名的，所以出错了，要匹配具体属性才行，比如”//input[contains(@id,’pass’)]”.

4.FindElement/FindElements:
FindElement()方法接受一些东西作为 Parameter/Argument 并且是By Object。By是用于在定位器值的帮助下在文档中定位元素的机制。By的正常语法如下所示：
IWebElement Account = driver.FindElement(By.ID("Account"));
或：
By Locator = By.ID("Account");
IWebElement Account = driver.FindElement(Locator);

FindElement/FindElements的区别：
FindElement()和FindElements()方法之间的区别是第一个返回一个WebElement对象，否则它会抛出异常，后者返回一个WebElements列表，如果没有 DOM 元素与查询匹配，它可以返回一个空列表。

5.学习Xpath定位元素位置

1.Selenium中的Xpath函数？
因为页面中几个不同的元素可能具有相同或相似的属性，所以有时候不仅仅是ID什么的无法定位，Xpath也有可能会返回多个值，所以这个时候要借用Xpath函数，帮助Xpath有效的标识唯一元素。

2.Xpath Contains()函数:
XPath Contains()是创建XPath表达式时使用的方法之一。如果任何属性的部分值动态变化，我们可以使用它它可以通过使用其部分值来识别任何属性。使用 XPath contains()方法的语法是：
tag_name[contains(@attribute,'value_of_attribute')]
其中，attribute为元素的标记属性，value是属性包含的部分值。

上图的Contans函数为：
//div[contains(@class,'login')]//button
或
//button[contains(.,'Log')]

3.XPath Starts-with()函数：
顾名思义，XPath starts-with()函数查找具有以特定给定字符或字符序列开头的属性值的元素。此功能在处理动态网页时非常有用。想象一个元素，其属性值随着每次页面加载或页面操作而不断变化。通常，这些动态元素很少有共同的起始字符，然后是随机的动态文本。除了动态属性，这也可以识别静态元素。
Starts-with()方法的语法是：
tag_name[starts-with(@attribute,'Part_of_Attribute_value')]

4.XPath text()函数：
此功能使用网页元素的文本来定位网页上的元素。如果您的元素包含文本，例如标签，则此功能非常有用，标签始终包含静态文本。
XPath中text()函数的语法是:
//tag_name[text()='Text of the element']
还是下图

//button[text()='Login']"
